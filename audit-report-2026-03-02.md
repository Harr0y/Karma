## 发现的问题

### P0 - 严重问题

#### 1. 工具调用信息暴露给用户

**严重程度**: P0 (严重)
**发现轮次**: 第 2 轮

**问题描述**:
在对话第 2 轮，当用户提供生辰信息后，Karma 调用了 webReader 工具查询农历转换，但工具调用的详细信息被直接暴露给了用户：

```
**🌐 Z.ai Built-in Tool: webReader**

**Input:**
```json
{"url":"https://wannianli.tianqi.com/yinli/1990-3-8.html"}
```

*Executing on server...*
```

**预期行为**:
工具调用应该是内部操作，不应该暴露给用户。

**影响**:
- 破坏了"命理师"的人设体验
- 暴露了系统内部实现细节
- 降低用户信任度

### P1 - 高优先级问题

#### 2. 回复内容截断问题

**严重程度**: P1 (高)
**发现轮次**: 第 12、14、15、16、17、18 轮

**问题描述**:
在多轮对话中，Karma 的回复出现明显的截断现象，只显示部分内容：

**第 12 轮**:
```
data: {"type":"text","content":"\n\n\n"}

data: {"type":"text","content":"始调理，别拖。\n\n还有别的想问的吗？"}
```
（明显缺少前面的内容）

**第 14 轮**:
```
data: {"type":"text","content":"\n\n"}

data: {"type":"text","content":"话，他们能高兴半天。\n\n你爸抽烟吗？"}
```

**第 15 轮**:
```
data: {"type":"text","content":"\n\n"}

data: {"type":"text","content":"另外他平时走路喘不喘？爬楼梯费劲吗？"}
```

**预期行为**:
回复应该完整显示，不应该出现截断。

**可能原因**:
- SSE 流式传输中的内容丢失
- Agent 输出被意外截断
- 消息拼接逻辑问题

**影响**:
- 用户无法获得完整的咨询内容
- 对话连贯性受损
- 用户体验严重下降

---

### P2 - 中优先级问题

#### 3. 农历日期解析失败无友好提示

**严重程度**: P2 (中)
**发现轮次**: 第 2 轮

**问题描述**:
当用户提供农历日期时（"1990年农历三月初八早上7点多"），八字排盘工具返回错误：
```
"toolUseResult": "Error: 八字排盘失败: Invalid birth date: 1990年农历三月初八早上7点多"
```

但 Karma 最终通过 webReader 工具手动查询解决了这个问题，用户没有感知到错误。

**预期行为**:
- 八字排盘工具应该支持农历日期解析
- 或者在失败时给出更友好的提示

**建议**:
- 增强 date-parsing skill 的农历解析能力
- 添加农历到公历的自动转换
