## 3. 实现计划

### Phase 9.1: 过滤器增强 (优先级最高)

**目标:** 过滤所有内部标签和工具调用展示

**文件:** `src/agent/monologue-filter.ts`

**改动:**

```typescript
// 当前只过滤 inner_monologue
// 需要增加过滤:
const FILTERED_PATTERNS = [
  /<inner_monologue>[\s\S]*?<\/inner_monologue>/g,
  /<client_info>[\s\S]*?<\/client_info>/g,
  /<prediction[^>]*>[\s\S]*?<\/prediction>/g,
];

// 在 SSE 输出时过滤 tool_use 类型消息
if (msg.type === 'tool_use') {
  // 不发送给用户
  continue;
}
```

**工作量:** 1 小时

---

### Phase 9.2: 回复截断修复 (P1)

**目标:** 调查并修复回复被截断的问题

**可能原因:**
1. SDK 流式返回时超时
2. 网络连接中断
3. 服务器端缓冲区问题

**调查步骤:**
1. 添加详细日志记录流式输出的每个 chunk
2. 检查 SDK 返回的完整消息
3. 验证 SSE 连接稳定性

**工作量:** 2-3 小时

---

### Phase 9.3: 信息收集优化

**目标:** 主动询问姓名，提升个性化

**改动:** 在 `GREETING_PROMPT` 或系统提示中增加姓名询问

**工作量:** 0.5 小时

---

### Phase 9.4: 重新测试验证

**目标:** 验证修复效果

1. 修复后重新运行 3 个 persona 测试
2. 确认平均分 ≥ 8.5
3. 确认无 P1 问题

**工作量:** 1 小时
