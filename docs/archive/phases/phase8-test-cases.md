### TC-001: 好奇青年 - 首次咨询事业运

#### 2.1.1 人设档案

```yaml
id: curious-young
name: 小明
age: 25
gender: male
occupation: 程序员
city: 深圳

# 生辰信息 (测试信息提取)
birthDate: "1998-05-15"          # 公历
birthTime: "14:30"               # 下午2:30
birthPlace: "湖南省长沙市"        # 出生地
lunarDate: "1998年四月二十"       # 农历 (验证历法转换)

# 八字 (预期提取结果)
bazi:
  year: "戊寅"
  month: "丁巳"
  day: "乙亥"
  hour: "癸未"

# 性格特点
personality:
  - 好奇心强，喜欢追问
  - 说话随意，用网络用语
  - 对命理了解很少，第一次接触
  - 有一定怀疑但愿意尝试

# 本次咨询
purpose: "想知道自己事业发展方向，最近工作压力大，考虑跳槽"
concerns: ["事业方向", "跳槽时机", "财运"]

# 对话风格
dialogueStyle:
  - 会用表情符号 😅 😂
  - 会问 "真的假的？"
  - 说话简短，不喜欢长篇大论
  - 可能中途会被打断
```

#### 2.1.2 预期对话流程

```
[Round 1]
小明: 师傅你好，听说你会算命？真的假的 😅

[Round 2] - Karma 应主动问候并引导
Karma: [问候] + [介绍] + [询问生辰]
小明: 额...我1998年5月15生的，下午两点半吧好像

[Round 3] - Karma 应确认信息并询问地点
Karma: [确认时间] + [询问出生地]
小明: 长沙，就是湖南那个长沙

[Round 4] - Karma 应确认信息完整，开始解读
Karma: [确认信息完整] + [开始解读或询问具体想问什么]
小明: 我就随便问问，主要想知道事业吧最近工作好累想辞职

[Round 5] - Karma 应展开事业解读
Karma: [事业运势解读]
小明: 哦哦那我是不是该跳槽？什么时候跳比较好？

[Round 6] - Karma 给出具体建议
Karma: [跳槽时机建议]
小明: 哈哈好的谢谢师傅，那财运呢？

[Round 7] - Karma 解答财运
Karma: [财运解读]
小明: 嗯嗯明白了，下次有问题再来问你

[Round 8] - Karma 结束语
Karma: [温暖结束语]
小明: 好的拜拜 👋

[END - 预计 8-10 轮]
```

#### 2.1.3 评估标准

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **信息提取** | | 10 | ≥8 |
| - 姓名或称呼 | 提取出一个称呼即可 | 2 | 2 |
| - 出生日期 | 正确提取 1998-05-15 | 2 | 2 |
| - 出生时间 | 正确提取 14:30 | 2 | 2 |
| - 出生地点 | 正确提取长沙 | 2 | 2 |
| - 咨询目的 | 识别出事业、跳槽 | 2 | 2 |

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **亲和力** | | 10 | ≥7 |
| - 问候温度 | 友好不生硬 | 2 | 2 |
| - 回应风格 | 适应对方随意风格 | 3 | 2 |
| - 共情表达 | 对工作压力表示理解 | 3 | 2 |
| - 结束温暖度 | 友好道别 | 2 | 1 |

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **专业度** | | 10 | ≥7 |
| - 八字准确性 | 正确推算八字 | 3 | 3 |
| - 解读逻辑 | 有理有据 | 3 | 2 |
| - 建议具体 | 给出具体时间/方向 | 2 | 1 |
| - 不夸大 | 不做绝对承诺 | 2 | 1 |

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **对话自然度** | | 10 | ≥7 |
| - 语言匹配 | 不过于正式 | 3 | 2 |
| - 回复长度 | 适中不啰嗦 | 3 | 2 |
| - 过渡流畅 | 话题转换自然 | 2 | 2 |
| - 无重复 | 不重复说相同的话 | 2 | 1 |

#### 2.1.4 期望通过的底线

- ✅ 信息提取完整度 ≥ 80% (4/5 项)
- ✅ 总分 ≥ 7.0/10
- ✅ 无 P0 问题 (严重错误、不当言论)
- ✅ 用户 (模拟) 表示愿意再次咨询

#### 2.1.5 失败案例标记

```yaml
P0_failures:
  - 输出不当言论 (宗教敏感、歧视等)
  - 八字推算明显错误 (如天干地支组合不合法)
  - 无法完成信息收集就给出结论
  - 回复为空或乱码

P1_failures:
  - 信息遗漏 (出生地、时间等)
  - 回复过于模板化，不够个性化
  - 跳过关键确认步骤
  - 给出绝对化承诺 ("一定会发财")

P2_failures:
  - 语气过于正式/老派
  - 回复过长 (超过 500 字)
  - 重复确认已获取的信息
  - 结束语不够温暖
```

```yaml
id: anxious-mom
name: 王女士
age: 42
gender: female
occupation: 全职妈妈
city: 上海

# 咨询对象信息 (注意：是咨询孩子，不是自己)
child:
  name: "小宝"
  birthDate: "2012-08-20"
  birthTime: "06:15"
  birthPlace: "上海市浦东新区"
  grade: "五年级"
  issue: "成绩下滑，不爱学习，沉迷手机"

# 性格特点
personality:
  - 非常焦虑，语速快
  - 说话可能跳跃，想到哪说哪
  - 对孩子期望高
  - 容易被情绪影响
  - 需要"定心丸"

# 本次咨询
purpose: "想知道孩子学习能不能好起来，要不要请家教，该不该转学"
concerns: ["孩子学业", "孩子未来", "教育决策"]

# 对话风格
dialogueStyle:
  - 说话快，信息量大
  - 会反复确认 "真的吗？"
  - 会问很多问题
  - 需要被安抚
```

#### 2.2.2 预期对话流程

```
[Round 1]
王女士: 师傅您好！我想问问我家孩子的事，他今年五年级了成绩一直在下滑我很担心不知道怎么办才好

[Round 2] - Karma 应安抚情绪，引导提供孩子生辰
Karma: [先安抚] + [询问孩子生辰]
王女士: 2012年8月20号生的，早上六点十五，在上海浦东，师傅您看看他这孩子是不是不是读书的料啊我很着急

[Round 3] - Karma 应温和回应，开始分析
Karma: [不要直接否定] + [分析学业运势]
王女士: 那要不要给他请个家教啊？还是换个学校？我听说有些学校抓得紧

[Round 4] - Karma 给出具体建议
Karma: [具体建议] + [时间节点]
王女士: 师傅您说他什么时候能好转？我现在天天睡不着觉

[Round 5] - Karma 情绪安抚
Karma: [安抚] + [给希望]
王女士: 哎您说得我心里踏实多了，那我就按您说的试试

[Round 6]
Karma: [总结] + [结束语]
王女士: 谢谢师傅，有问题我再来问您

[END - 预计 6-8 轮]
```

#### 2.2.3 评估标准

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **信息提取** | | 10 | ≥8 |
| - 咨询对象 | 正确识别是咨询孩子 | 2 | 2 |
| - 孩子生辰 | 完整提取 | 4 | 4 |
| - 问题焦点 | 识别学业、手机问题 | 2 | 1 |
| - 决策诉求 | 识别家教、转学问题 | 2 | 1 |

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **亲和力** | | 10 | ≥8 |
| - 情绪识别 | 识别出焦虑情绪 | 2 | 2 |
| - 安抚时机 | 适时安抚，不急不缓 | 3 | 2 |
| - 共情深度 | 真正理解母亲担忧 | 3 | 2 |
| - 给予希望 | 不只是分析，还给信心 | 2 | 2 |

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **专业度** | | 10 | ≥7 |
| - 对象区分 | 明确分析孩子而非母亲 | 2 | 2 |
| - 学业分析 | 基于八字分析学业宫 | 3 | 2 |
| - 建议具体 | 具体的行动建议 | 3 | 2 |
| - 时间预判 | 给出好转时间窗口 | 2 | 1 |

| 维度 | 评分标准 | 满分 | 期望分数 |
|------|----------|------|----------|
| **对话自然度** | | 10 | ≥7 |
| - 语速匹配 | 适应快节奏 | 2 | 1 |
| - 信息过滤 | 帮助梳理跳跃信息 | 3 | 2 |
| - 不催促 | 不催促对方 | 2 | 2 |
| - 总结清晰 | 帮忙总结要点 | 3 | 2 |

#### 2.2.4 核心测试点

1. **情绪识别** - 能否识别用户焦虑情绪并优先安抚
2. **对象区分** - 明白咨询的是孩子，不是本人
3. **信息过载处理** - 用户一次说很多信息，能否有效提取
4. **避免说教** - 不指责家长，不制造更多焦虑

### TC-003: 怀疑论者 - 质疑八字准确性

#### 2.3.1 人设档案

```yaml
id: skeptic
name: 老张
age: 55
gender: male
occupation: 退休工程师
city: 北京

# 生辰信息
birthDate: "1969-03-12"
birthTime: "子时"              # 故意说得模糊
birthPlace: "北京"
lunarDate: "1969年正月二十四"

# 性格特点
personality:
  - 理性、逻辑性强
  - 对命理持怀疑态度
  - 会追问原理
  - 可能会质疑
  - 不容易被说服

# 本次咨询
purpose: "朋友推荐来的，想试试到底准不准"
concerns: ["验证准确性", "退休后的规划"]

# 对话风格
dialogueStyle:
  - 直来直去
  - 会问"你怎么知道的？"
  - 会说"这个我查过，不是这样"
  - 冷静但不热情
```

#### 2.3.2 预期对话流程

```
[Round 1]
老张: 朋友说你算得准，我来看看

[Round 2]
Karma: [友好回应] + [询问生辰]
老张: 1969年3月12号，子时生的，北京

[Round 3] - Karma 确认信息
Karma: [确认] + [说明需要更具体时间或用子时推算]
老张: 你们不是能算吗？子时就是半夜吧，还要多精确？

[Round 4] - Karma 专业解释 + 开始分析
Karma: [解释子时范围] + [给出推算结果]
老张: 你说我属土，我查了下1969年是己酉年，应该是属土没错。那你怎么知道我是什么命？

[Round 5] - Karma 专业论证
Karma: [解释纳音/日主等]
老张: 行，你说我最近有变动？我去年刚退休，这算变动吗？

[Round 6]
Karma: [对应解读] + [深入分析]
老张: 那你看看我退休后做什么合适？我不爱跳广场舞

[Round 7]
Karma: [建议]
老张: 嗯，有点道理，我再想想

[END - 预计 7-9 轮]
```

#### 2.3.3 核心测试点

1. **应对质疑** - 不回避，不生气，专业回应
2. **专业解释** - 能解释八字的推算原理
3. **不争辩** - 不与用户争辩对错
4. **实际对应** - 分析结果能与用户实际对应

#### 2.3.4 特别注意

```yaml
must_avoid:
  - "你不懂这个很正常" # 避免居高临下
  - "信则有不信则无" # 避免推脱
  - 与用户辩论对错
  - 对质疑表现出不耐烦

should_do:
  - 承认命理有局限性
  - 用事实说话，不强求相信
  - 对专业问题耐心解释
  - 保持开放态度
```

### TC-004: 回头客 - 老客回访问感情

#### 2.4.1 人设档案

```yaml
id: returning-customer
name: 李姐
age: 35
gender: female

city: 广州

# 生辰信息
birthDate: "1989-11-08"
birthTime: "19:45"
birthPlace: "广东省广州市"

# 历史 (上次咨询记录)
previousSession:
  date: "2024-01-15"
  topic: "事业发展"
  extractedInfo:
    name: "李小姐"
    concerns: ["工作晋升", "换工作"]
  summary: "当时建议春季有变动机会"

# 性格特点
personality:
  - 熟悉咨询流程
  - 会提起上次内容
  - 期待被记住
  - 问题更深入

# 本次咨询
purpose: "上次说的事业应验了，这次来问感情"
concerns: ["感情婚姻", "桃花运"]

# 对话风格
dialogueStyle:
  - 友好亲切
  - 会说"师傅上次你说的挺准的"
  - 期待个性化回应
```

#### 2.4.2 预期对话流程

```
[Round 1]
李姐: 师傅好久不见！上次您说的那个工作机会真的来了，我这次来问点别的

[Round 2] - Karma 应调出历史记录，认出老用户
Karma: [友好回应] + [提及上次内容]
李姐: 对对对，就是我，这次我想问问感情，我一直单身

[Round 3]
Karma: [过渡到感情话题] + [询问或确认生辰]
李姐: 生辰您那儿应该有吧？1989年11月8号晚上七点四十五

[Round 4]
Karma: [确认] + [感情运势分析]
李姐: 那你说我什么时候能遇到对的人啊，家里催死了

[Round 5]
Karma: [桃花运分析] + [时间节点]
李姐: 真的吗？那我该注意什么？上次您说的注意事项还挺有用的

[Round 6]
Karma: [具体建议]
李姐: 好的谢谢师傅，有问题再来

[END - 预计 6-7 轮]
```

#### 2.4.3 核心测试点

1. **历史记录** - 能否调出并利用历史信息
2. **个性化** - 不要像对待新用户一样
3. **连贯性** - 与上次咨询有关联
4. **过渡自然** - 从事业过渡到感情

#### 2.4.4 特别注意

```yaml
memory_test:
  - 是否记录了上次咨询的基本信息？
  - 能否主动提及"上次你问的是事业"？
  - 能否利用上次建立的关系？

continuity_test:
  - 不需要重新问已经问过的基本信息
  - 可以参考上次的八字推算结果
  - 语气更熟悉，不像陌生人
```

```
阿强: 师傅你好，我想看个完整的八字命盘，事业财运感情健康都说说
```

```yaml
  - 说话直接
  - "别的不用说了"
  - "你就告诉我不行还是行"
  - 可能只问一两轮就走
```

#### 2.6.2 预期对话流程

```
[Round 1]
小刘: 师傅我就问一个事，明天签合同行不行，我1996年7月22号中午11点成都生的

[Round 2] - Karma 应快速回应核心问题
Karma: [直接回答] + [简要理由]
小刘: 行，那就明天签，谢了

[END - 仅 2 轮]
```

#### 2.6.3 核心测试点

1. **响应速度** - 不绕弯子，快速回答
2. **简洁性** - 回答精炼，不啰嗦
3. **核心明确** - 给出明确建议 (行/不行)
4. **适应风格** - 不强迫用户走完整流程

## 3. 评估体系详解

### 3.1 评分维度定义

#### 3.1.1 信息提取 (Info Extraction)

```typescript
interface InfoExtractionScore {
  // 满分 10 分
  score: number;  // 0-10

  // 各字段得分
  fields: {
    name: { extracted: boolean; score: number };        // 2分
    birthDate: { extracted: boolean; score: number };   // 2分
    birthTime: { extracted: boolean; score: number };   // 2分
    birthPlace: { extracted: boolean; score: number };  // 2分
    purpose: { extracted: boolean; score: number };     // 2分
  };

  // 提取准确性
  accuracy: {
    correctFields: string[];
    incorrectFields: string[];
    missingFields: string[];
  };
}
```

**评分规则：**
- 正确提取: +2分/字段
- 部分提取: +1分/字段 (如只提取年份没提取具体日期)
- 错误提取: 0分
- 未尝试提取: 0分

**特殊情况：**
- TC-006 (急躁型): 用户直接提供信息，算自动提取成功
- TC-007 (模糊型): 用户不提供，评估引导能力而非提取结果

#### 3.1.2 亲和力 (Warmth)

```typescript
interface WarmthScore {
  // 满分 10 分
  score: number;

  // 子维度
  dimensions: {
    greeting: number;        // 问候温度 (0-2)
    adaptation: number;      // 风格适应 (0-3)
    empathy: number;         // 共情能力 (0-3)
    closing: number;         // 结束温暖度 (0-2)
  };

  // 证据
  evidence: {
    positive: string[];      // 体现亲和力的片段
    negative: string[];      // 缺乏亲和力的片段
  };
}
```

**评分依据：**

| 子维度 | 高分表现 | 低分表现 |
|--------|----------|----------|
| 问候 | 亲切自然，有温度 | 生硬，模板化 |
| 风格适应 | 根据用户风格调整 | 千篇一律 |
| 共情 | 理解并回应情绪 | 只关注信息 |
| 结束 | 温暖道别，留有余地 | 突然结束 |

#### 3.1.3 专业度 (Professionalism)

```typescript
interface ProfessionalismScore {
  score: number;

  dimensions: {
    accuracy: number;        // 八字准确性 (0-3)
    logic: number;           // 解读逻辑性 (0-3)
    specificity: number;     // 建议具体性 (0-2)
    caution: number;         // 不夸大承诺 (0-2)
  };

  evidence: {
    accurateBazi: string;    // 正确的八字推算
    logicalReasoning: string[]; // 有逻辑的推论
    specificAdvice: string[];  // 具体建议
    cautionaryPhrases: string[]; // 谨慎表述
  };
}
```

**专业度红线：**
- ❌ 八字推算明显错误 (天干地支不合法)
- ❌ 做绝对承诺 ("一定会..."、"保证...")
- ❌ 迷信说法 (如"必须买某物化解")
- ❌ 超出命理范围的建议 (医疗、法律等)

#### 3.1.4 用户满意度 (User Satisfaction)

```typescript
interface SatisfactionScore {
  score: number;

  basedOn: {
    willingnessToReturn: boolean;   // 是否愿意再来
    recommendationLikelihood: number; // 推荐意愿 (0-10)
    overallFeeling: string;         // 整体感受描述
  };

  reasoning: string;  // 评分理由
}
```

**评分依据：**
- 9-10分: 非常满意，主动表示要推荐
- 7-8分: 满意，愿意再次咨询
- 5-6分: 一般，可以接受
- 3-4分: 不太满意，有改进空间
- 0-2分: 很不满意

#### 3.1.5 对话自然度 (Naturalness)

```typescript
interface NaturalnessScore {
  score: number;

  dimensions: {
    languageMatch: number;   // 语言风格匹配 (0-3)
    length: number;          // 回复长度适中 (0-3)
    transition: number;      // 过渡流畅 (0-2)
    noRepetition: number;    // 无重复内容 (0-2)
  };

  issues: {
    tooFormal: string[];     // 过于正式的表述
    tooLong: string[];       // 过长的回复
    abrupt: string[];        // 突兀的转换
    repetitive: string[];    // 重复的内容
  };
}
```

**自然度问题示例：**
- "根据您的八字命盘分析..." → 太正式
- 连续 500+ 字回复 → 太长
- 突然跳到新话题 → 过渡生硬
- "您的生辰是？" x3 → 重复

### 3.2 综合评分计算

```typescript
function calculateOverallScore(metrics: EvaluationMetrics): number {
  const weights = {
    infoExtraction: 0.20,    // 20%
    warmth: 0.20,            // 20%
    professionalism: 0.25,   // 25%
    userSatisfaction: 0.20,  // 20%
    naturalness: 0.15        // 15%
  };

  return (
    metrics.infoExtraction.score * weights.infoExtraction +
    metrics.warmth.score * weights.warmth +
    metrics.professionalism.score * weights.professionalism +
    metrics.userSatisfaction.score * weights.userSatisfaction +
    metrics.naturalness.score * weights.naturalness
  );
}
```

### 3.3 问题分级标准

```yaml
P0_critical:
  description: "严重问题，必须立即修复"
  examples:
    - 输出敏感/不当内容
    - 核心功能无法使用
    - 八字推算完全错误
  action: "阻断发布，立即修复"

P1_major:
  description: "主要问题，影响用户体验"
  examples:
    - 关键信息遗漏
    - 回复与问题不相关
    - 给出不当建议
  action: "优先修复"

P2_minor:
  description: "次要问题，可以后续优化"
  examples:
    - 语气不够自然
    - 回复略长
    - 过渡不够平滑
  action: "记录并优化"

P3_suggestion:
  description: "建议改进，锦上添花"
  examples:
    - 可以更个性化
    - 可以更有趣
  action: "有空优化"
```
